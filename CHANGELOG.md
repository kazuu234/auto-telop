# Changelog

## v1.0.5

### 修正

- **エディタのスタイルプレビューの文字サイズをFCPの実際の描画に一致させました** — プレビューが常に約33%大きく表示されていた係数を撤去（実測でFCP側6.85%・プレビュー6.83%に一致）。縁取りの太さもプレビュー縮尺に連動するようにし、FCPのBasic Titleと同様にプレビューでも文字を折り返さないようにしました。
- **画面写真つきセットアップガイドを公開** — https://kazuu234.github.io/auto-telop/install_guide.html （READMEとアプリ内ガイドからもリンク）

## v1.0.4

### 修正

- **文字起こし中にもう1つのGUIウインドウが立ち上がる問題を修正** — Whisper（torch）が処理中に生成するワーカープロセスが、PyInstaller化されたアプリ本体を再実行してしまうのが原因でした。エントリポイントに `multiprocessing.freeze_support()` を追加して解消。
- **セットアップ手順を macOS 15以降に対応** — 「右クリック →開く」バイパスの廃止に伴い、「初回セットアップ.command」は**システム設定 → プライバシーとセキュリティ →「このまま開く」**で承認する手順に更新しました（ガイド・README。macOS 14以前の手順も併記）。

## v1.0.3

**テロップがFCPで表示されない問題を修正しました。**（実際のFinal Cut Proでの診断・実機確認済み）

### 修正

- **フォント未インストール時にテロップが不可視になる問題を修正** — 指定フォント（Noto Sans JP / Inter）が未インストールだと、FCPはスタイル全体を破棄して6ptで表示するため実質見えなくなっていました。書き出し時にフォントの有無を確認し、無ければmacOS標準の **ヒラギノ角ゴシック W7 / Helvetica Neue Bold** に自動フォールバックするようにしました。
- **テロップ位置が画面外に飛ぶ問題を修正** — FCPXMLの `adjust-transform position` は「画面サイズに対する%」ですが、ピクセルに変換して出力していたため画面外に配置されていました。%をそのまま出力するよう修正（デフォルト `position_y: -40` ≒ 画面下から10〜14%）。
- **自動生成されるFCPXMLにスタイルが適用されない問題を修正** — 抽出直後の `_pipeline.fcpxml` がスタイル無し（Helvetica 48px・位置なし）で生成されていました。
- エディタのスタイルプレビューの位置換算をFCPの実挙動に一致させました。

### フォントについて

Noto Sans JP / Inter を使いたい場合は Google Fonts からインストールしてください（READMEに手順あり）。未インストールでもヒラギノで正常に表示されます。

## v1.0.2

**「ffmpeg が見つからない（No such file or directory）」を修正しました。**

### 修正

- **Finderからの起動時に ffmpeg / ffprobe を見つけられない不具合を修正** — Finder（.app）から起動したアプリはターミナルの `PATH` を引き継がないため、Homebrewで入れた `ffmpeg`（`/opt/homebrew/bin` など）が見つからず、抽出が `No such file or directory: 'ffmpeg'` で失敗していました。起動時に主要なインストール先を `PATH` に追加するようにしました（`/opt/homebrew/bin`, `/usr/local/bin`, `/opt/local/bin` など）。
  - 引き続き、動画の抽出には `ffmpeg` が必要です。未インストールの場合は `brew install ffmpeg` を実行してください。

## v1.0.1

**「壊れている」問題と、ファイル選択が効かない不具合を修正しました。**

### 修正

- **「壊れているため開けません」への対処を同梱** — 配布zipに `初回セットアップ.command` を同梱しました。ダウンロード後にこれを実行（右クリック →「開く」）すると、未署名アプリの検疫属性（`com.apple.quarantine`）を自動で取り除き、アプリが起動できるようになります。
- **「動画を選択」やドラッグ&ドロップが反応しない不具合を修正** — ネイティブ判定（`window.pywebview.api`）をスクリプト読み込み時に固定していたため、アプリ版でもブラウザ扱いになりファイル選択ダイアログが開かないことがありました。呼び出し時に毎回判定するよう修正しました。

### 使い方（アップデート）

1. リリースから `AutoTelop-mac.zip` をダウンロードして展開
2. `AutoTelop.app` をアプリケーションフォルダへ移動
3. 同梱の `初回セットアップ.command` を**右クリック →「開く」**で実行
4. `AutoTelop.app` を起動 → 「動画を選択」またはドラッグ&ドロップで動画を追加

## v1.0.0

**Auto Telop が Mac用のGUIアプリになりました。** ターミナルもブラウザも不要です。

### 新機能

- **GUIローカルアプリ（PyWebView）** — アプリを起動して動画を扱えます。既存の校閲エディタをアプリ内に内蔵。
- **ライブラリ画面** — 動画をドラッグ&ドロップ（複数可）または「＋ 動画を選択」で追加。
  - 抽出が終わると、動画と文字起こし結果をまとめた**エントリー**が一覧に追加されます。
  - エントリーをクリックすると校閲画面に移動します。
  - 複数動画は**1本ずつ順番に**処理され、進捗が表示されます。
  - アプリを閉じて再度開いても、**以前のエントリーはそのまま残ります**（`projects/` に保存、DB不要）。
  - エントリーの削除に対応。
- **保存先の指定** — 保存時にネイティブのダイアログで**出力先フォルダ・ファイル名を選べます**。前回選んだフォルダを記憶します。
- **バージョン管理とアプリ内アップデート** — 「更新を確認」ボタンから GitHub Releases の最新版を確認し、ダウンロードできます。バージョンは `VERSION` で管理。

### 開発・配布

- `desktop.py` … PyWebViewランチャー（Flaskを内部起動＋ネイティブダイアログ連携）
- `jobs.py` … 抽出ジョブの順次処理キュー
- `updater.py` / `version.py` … 更新確認・ダウンロード
- `AutoTelop.spec` / `build_macos.sh` … PyInstallerによる `.app` ビルド
- README・セットアップガイド（`guide.html`）をGUIアプリ向けに刷新

### 補足

- Whisperのモデルは初回起動時に自動ダウンロードされます（約1.5GB）。
- FFmpeg（`brew install ffmpeg`）が必要です。アプリには同梱されません。
- クロスプラットフォーム構成のため、Windows版（`.exe`）は今後追加しやすくなっています。
